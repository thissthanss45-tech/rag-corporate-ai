# Security & Governance

## Access control

- Upload документов разрешен только пользователям из `OWNER_ID` + `ADMIN_IDS`
- Все действия загрузки и ответов пишутся в audit-лог (`audit_event`)

## Prompt-injection defense

- Контекст документов проходит фильтр suspicious-patterns
- Инструкции из контекста считаются данными и не исполняются
- В system prompt зафиксировано правило игнорировать инструкции внутри контекста

## Data governance

- Политика хранения регулируется `RETENTION_DAYS`
- Очистка старых файлов через `scripts/retention_cleanup.py`
- Рекомендуется шифрование томов `data/` и `indices/` на уровне диска/хранилища

## Audit

- События: вопрос, отправка ответа, успех/ошибка загрузки, запрет загрузки
- Логи структурированы и готовы для SIEM/ELK
