# Руководство пользователя RAG (Telegram)

## 1) Быстрый старт (1 минута)
1. Откройте бота и нажмите `/start`.
2. Нажмите **📄 Загрузить документ** и отправьте файл.
3. Дождитесь сообщения `completed`.
4. Нажмите **🧠 Задать вопрос**, выберите модель и задайте вопрос.

Поддерживаемые форматы документов:
- PDF
- DOCX
- XLSX
- TXT
- RTF

Поддерживаемые голосовые вопросы:
- OGG

---

## 2) Главное меню
В меню доступны:
- **🧠 Задать вопрос** — начать диалог по документам.
- **🔁 Сменить модель** — переключить Llama/DeepSeek.
- **📄 Загрузить документ** — добавить новый документ в базу знаний.

---

## 3) Как загрузить документ
1. Нажмите **📄 Загрузить документ**.
2. Отправьте PDF/DOCX/XLSX/TXT/RTF.
3. Бот вернёт `task_id`.
4. Дождитесь статуса `completed`.

Если статус долго не меняется:
- подождите 1–3 минуты;
- отправьте файл повторно;
- передайте администратору `task_id`.

---

## 4) Как задавать вопросы
1. Нажмите **🧠 Задать вопрос**.
2. Выберите модель: **🦙 Llama** или **🐋 DeepSeek**.
3. Отправьте вопрос текстом.
4. Получите ответ и список источников.

Чтобы сменить модель в любой момент:
- нажмите **🔁 Сменить модель**,
- выберите другую модель,
- задайте следующий вопрос.

---

## 5) Голосовые вопросы
1. Нажмите **🧠 Задать вопрос**.
2. Выберите модель.
3. Отправьте голосовое сообщение (OGG).
4. Бот распознает речь и вернёт ответ.

Если распознавание плохое:
- говорите короче и чётче;
- запишите сообщение в тишине;
- при необходимости отправьте вопрос текстом.

---

## 6) Статусы обработки
- `queued` — задача в очереди.
- `processing` — файл обрабатывается.
- `completed` — файл успешно добавлен.
- `failed` — ошибка обработки.

---

## 7) Частые проблемы
**Неподдерживаемый формат**
- Используйте только PDF/DOCX/XLSX/TXT/RTF.

**Файл слишком большой**
- Разделите на части.

**Нет ответа от сервиса**
- Повторите запрос через 30–60 секунд.

**Ответ не по теме**
- Уточните вопрос (период, отдел, документ, процесс).

---

## 8) Безопасность
- Не отправляйте пароли, токены, приватные ключи.
- Не загружайте запрещённые к передаче документы.
- Проверяйте критичные ответы по исходным документам.

---

## 9) Обратная связь администратору
Если есть ошибка, передайте:
- время проблемы;
- действие (загрузка/вопрос/голос);
- `task_id` (если есть);
- короткое описание шага, где возникла проблема.
